Selection Structures in Python

A selection structure is used wherever a program needs different processing, or different output, based on the input or the needs of the user at run time. Selection is sometimes referred to as branching because one of two or more branches is used based on the results of a conditional statement. We have seen this before in the header of a while loop. The branch choice is to either continue in the loop or exit the loop based on the results of the conditional statement.

# Using the while loop to demonstrate branching
# ICS2O Unit 4, Activity 6
# Created by R. Kuzmochka
# Edited by K. Spindler

# Generate random numbers: seed() ensures they are actually random
import random
random.seed()

# Variables
userContinue = 'y'
userAnswer = 0

# Program Title
print ("Addition Questions Program")

# User can plan as many times as they like
while userContinue == 'y' or userContinue == 'Y'::
    # Create the numbers for the addition question
    num1 = random.randint(1, 10)
    num2 = random.randint(1, 10)

    # exit branches out of the second loop when the answer is correct
    while userAnswer != num1 + num2:
        # User Prompt
        print ("Answer the addition question: ")
        print (num1, " + ", num2, " = ", end='')

        # Input: User's response
        userAnswer = int(input())

    # exit branches out of the first loop if user entered 'n'
    userContinue = input("Continue? (y/n)")

The If Statement

In the Python language, a selection structure is created through the use of the if ... statement. There are a number of variations on the if ... statement, such as if ... else and if ... elif ... else. Each variation creates a different number of branches. Other languages (like Java & C++) have more options for selection structures (such as case or switch statements).

Simple (One Branch) Selection (if ...)

The most basic type of selection has a single, temporary branch. The branch is selected if a condition is met. The syntax of this statement is a follows:

if <condition>:
    # code to run when condition evaluates to True

The condition is the same type of condition used for the headers of while loops. If the condition is True then the programming code in the selection structure is executed. If the condition is False then all of the code in the selection structure is ignored.

The following is an example of pygame animation using the if .. statement to detect if the ball has hit the top or bottom, left or right side of the screen. It uses the variables dx and dy to store the change in position of the ball horizontally and vertically.

When the ball reaches the side of the screen, the sign (+/-) of dx or dy is changed. This will make the ball appear to 'bounce' off the sides of the screen. Try running this program in Eclipse

'''
Created: Aug 2018
BallBounce.py
pygame animation using selection statements
@author: K. Spindler
'''
import pygame
import random

from pygame import Color
from pygame import draw
from pygame import display
from pygame import time

# Constants
SCREEN_WIDTH = 600
SCREEN_HEIGHT = 400
RADIUS = 30

# Variables

# position of ball
center_x = 250
center_y = 250

# change in position of the ball
dx = random.randint(2, 4)
dy = random.randint(2, 4)

# initialize pygame modules
pygame.init()

clock = time.Clock()

# get a surface for graphics display
gameDisplay = display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))

while True:
    # white background
    gameDisplay.fill(Color('white'))

    # draw the ball
    draw.circle(gameDisplay, Color('purple'), (center_x, center_y), RADIUS)

    # show the graphics on the screen
    display.flip()

    # check for ball hitting the sides; make it bounce
    if center_x + RADIUS > SCREEN_WIDTH or center_x - RADIUS < 0:
        dx = -dx

    # check for ball hitting the top or bottom; make it bounce
    if center_y + RADIUS > SCREEN_HEIGHT or center_y - RADIUS < 0:
        dy = -dy

    # get ready for next frame - move the ball
    center_x += dx
    center_y += dy

    #delay the program to obtain 45 frames per second
    clock.tick(45)

In the above example notice that the if ... statement is used to change the value of a variable only when it is needed (ie the ball hits the sides of the screen). If the change is not needed dx or dy will stay the same and the ball will continue moving in the same direction.

Two Branch Selection (if ... else ...)

Programming problems are often solved by two or more options that are dependent on the circumstances in the program at the time. This two branch system is a simple extension of the if ... statement. This approach states that if the first option is not used then the second must be used. This is the if ... else structure.

The syntax of this statement is a follows:

if <condition>:
    # code to run when condition evaluates to True
else:
    # code to run when condition evaluates to False

An example of the use of this structure is below.

# Example of the if ... else ... statement, using two branches.
# This program evaluates whether a person is of driving age.
# ICS2O Unit 4, Activity 6
# Created by R. Kuzmochka
# Edited by K. Spindler

# Constants
DRIVING_AGE = 16

# Input
userAge = int(input("Please enter your age: "))

# Processing and Output
if userAge >= DRIVING_AGE:
    # user's age is greater than the driving age
    print ("You can drive!")
else:
    # user's age must be less than driving age
    print ("You are too young to drive!")
Multi Branch Selection (if ... elif ... else ...)

While the two branch structure is very useful there are time when many branches are necessary. In these situations it is necessary to use a different structure using the elif keyword to start a new conditional branch. You can have as many elif statements as required. The syntax for this structure is as follows:

if <condition1>:
    # code to run when condition1 is True
elif <condition2>:
    # code to run when condition1 is False and condition2 is True
elif <condition3>:
    # code to run when condition1 and condition2 are False and condition3 is True
else:
    # code to run when all previous conditions are False

The elif permits the use of multiple conditions. The first statement that matches the condition will be implemented, so the order of these statements is very important! The condition is the same as that used in the header of while loops and in the if ... statements. The use of the else at the end is optional. Review the following example:

# Example of the if ... elif ... else ... statement, using 3 branches
# This program evaluates if a person is of driving age or needs a retest
# Seniors over 80 require a retest.
# ICS2O Unit 4, Activity 6
# Created by R. Kuzmochka
# Edited by K. Spindler

# Variables
DRIVING_AGE = 16
RETEST_AGE = 80

# Input
userAge = int(input("Please enter your age: "))

# Processing and Output
if userAge >= DRIVING_AGE and userAge < RETEST_AGE:  #notice the change
    # user's age is greater than the driving age, but not a senior
    print ("You can drive!")
elif userAge >= RETEST_AGE:
    # user is a senior and needs to retest for safety reasons
    print ("You will need a retest in order to drive.")
else:
    # user's age must be less than driving age
    print ("You are too young to drive!")
Nested Selection

Selection can be nested. This mean that selection can be included in another repetition or selection structure. However, when a selection structure is inside inside another selection structure there are several considerations. First, this nested structure will only be activated if the outer selection is True. This means that the selection is a secondary selection.

An example of a nested selection would be the handling of pygame events. Events allow a user to interact with the pygame program.

Add the following code to the BallBounce.py, just after the infinite loop header (while True:) This code allows the user to interact with the program. 

    # Main Event Loop - get the current event
    for evt in pygame.event.get():

        # Does the user want to quit? (red "x" button on upper right)
        if evt.type == pygame.QUIT:
            quit()

        # make ball respond to keyboard input
        if evt.type == pygame.KEYDOWN:
            if evt.key == pygame.K_DOWN:
                dy = random.randint(2, 4)

            elif evt.key == pygame.K_UP:
                dy = -random.randint(2, 4)

            elif evt.key == pygame.K_LEFT:
                dx = -random.randint(2, 4)

            elif evt.key == pygame.K_RIGHT:
                dx = random.randint(2, 4)

This code uses a special type of for loop. Instead of counting numbers, it cycles through the list of events that have been triggered since the last loop. This gives the user a chance to interact with the program, though mouse clicks or key strokes.

There are two main events that this program will detect:

  1. a QUIT event (triggered by the user clicking the red 'x' at the upper right corner of the window)
  2. KEYDOWN events (to respond when the user type a key on the keyboard)
    • When the user types an arrow key, the ball will respond by moving in that direction.
    • A random number (2, 3 or 4) will be used for the dx or dy variable, to give some variance in the movement
    • Note that the multi-branch if statement only executes when a KEYDOWN event has already been detected

Pseudocode

Pseudocode is the use of everyday language to describe the operations that the programmer sees as necessary to create a program or solve a problem. Pseudocode does not use programming language, but it does use programming concepts. The programmer essentially lays out the required variables and algorithms needed for the program before selecting a programming language. It requires that the programmer know the kind of operations that can be performed in a programming language.

Python, C, C++, Turing, and Java and many other programming languages use the same basic structures (sequence, repetition, selection). What can be created with a knowledge of Python commands can be used in other languages such a Turing, C, C++, C# and even Java. Additionally, the pseudocode demonstrates more clearly than the actual code the algorithms that the programmer sees as critical to solving the problem.

As an example, consider writing a program that would find the range of marks in a class, which would be the difference between the lowest and the highest mark. The marks are not sorted, so the code must read through the entire list of marks, keeping track of which mark is the lowest and the highest. Assume all marks are in percents, so between 0 and 100. The pseudocode would look like the following:

declare a variable highestMark, set to 0
declare a variable called lowestMark, set to 100
declare a variable called done, set to "N"
declare a variable currentMark, to hold the mark the user enters
declare a variable markRange, to hold that final output

while user is still entering marks
    prompt user for mark
    assign currentMark to user input

    if currentMark is less that lowestMark, we have a new low mark
        assign lowestMark to currentMark

    if currentMark is greater than highestMark, we have a new high mark
        assign highestMark to currentMark

    prompt user for Y or N to indicate if they are done entering marks
    assign to done
    back to start of loop

assign markRange to highestMark - lowestMark
output markRange to user

In Python, this code would look like this:

# Mark Range Calculator
# This program will determine the range of a set of marks
# provided by the user
# ICS2O Unit 4, Activity 6
# Created by K. Spindler

# Variables
highestMark = 0
lowestMark = 100
done = 'N'

# Loop while user is still entering marks
while done == 'N' or done == 'n':
    # Input: ask user for mark
    currentMark = int(input("Input a mark: "))

    #Processing
    if currentMark < lowestMark:
        # we have a new lowest mark
        lowestMark = currentMark

    if currentMark > highestMark:
        # we have a new highest mark
        highestMark = currentMark

    # prompt the user to continue or end the loop
    done = input("done marks? Y or N: ")

# Range is the difference between the highest and lowest mark
markRange = highestMark - lowestMark

# Output, newline for readability
print("\nDifference in marks is ", markRange)

In another language (Turing), the code for this program would be:

% Mark Range Calculator
% This program will determine the range of a set of marks
% provided by the user
% Created by K. Spindler

% Declaring Variables
var highestMark, lowestMark, currentMark, markRange : int
var done : string

% Initializing Variables
highestMark := 0
lowestMark := 100

loop
    % Input: ask user for mark
    put "Input a mark"
    get currentMark

    % Processing
    if (currentMark < lowestMark) then
        % we have a new lowest mark
        lowestMark := currentMark
    end if

    if (currentMark > highestMark) then
        % we have a new highest mark
        highestMark := currentMark
    end if

    % Prompt user to continue or end the loop
    put "done marks? Y or N"
    get done

    % End loop when user is done
    exit when done = "Y" or done = "y"
end loop

% Range is the difference between the highest and lowest mark
markRange := highestMark – lowestMark

% Output, newline for readability
put "\nDifference in marks is ", markRange

And here's what it would look like in Java:

import java.io.*;

/**
 * Mark Range Calculator
 * This program will determine the range of a set of marks
 * provided by the user
 * @author K. Spindler
 */
public class MarkCalculator {

    public static void main(String[] args) throws IOException {
        // Declaring Variables
        int highestMark, lowestMark, currentMark, markRange;
        char done;
        BufferedReader br;

        // Initializing variables
        highestMark = 0;
        lowestMark = 100;
        done = 'N';
        br = new BufferedReader(new InputStreamReader(System.in));

        // Loop while user is still entering marks
        while (done == 'N' || done == 'n') {
            // Input: ask user for mark
            System.out.print("Input a mark: ");
            currentMark = Integer.parseInt(br.readLine());

            /// Processing
            if (currentMark < lowestMark) {
                // we have a new lowest mark
                lowestMark = currentMark;
            }
            if (currentMark > highestMark) {
                // we have a new highest mark
                highestMark = currentMark;
            }
            // prompt the user to continue or end the loop
            System.out.print("Done marks? Y or N: ");
            done = br.readLine().charAt(0);
        }
        // Range is the difference between the highest and lowest mark
        markRange = highestMark - lowestMark;

        // Output, newline for readability
        System.out.print("\nDifference in marks is " + markRange);
    }
}

Also notice how much of the information in the pseudocode becomes comments in the actual code. This is one of the benefits to using pseudocode; your comments write themselves!
